apiVersion: v1
kind: ConfigMap
metadata:
  name: ldap-global-conf
  namespace: core-system
  labels:
    type: ldap
  annotations: (1)
    reflector.v1.k8s.emberstack.com/reflection-allowed-namespaces: "kube-system,default,core-system,core-middleware,core-app,share-app,monitor-app"
    reflector.v1.k8s.emberstack.com/reflection-allowed: "true"
    reflector.v1.k8s.emberstack.com/reflection-auto-enabled: "true"
    reflector.v1.k8s.emberstack.com/reflection-auto-namespaces: "kube-system,default,core-system,core-middleware,core-app,share-app,monitor-app"
data:
  ldap_org: "Dragon's Zone Team" (2)
  ldap_domain: 'd7z.net' (3)
  ldap_host: 'service-ldap.core-system.svc.cluster.local' (4)
  ldap_url: 'ldap://service-ldap.core-system.svc.cluster.local:389' (5)
  ldap_base_dn: 'dc=d7z,dc=net' (6)
  ldap_user_dn: 'ou=people,dc=d7z,dc=net' (7)
  ldap_group_dn: 'ou=groups,dc=d7z,dc=net' (8)
其中：
1 复制此 ConfigMap 到各个 namespace 下 （reflector 提供）
2 LDAP 配置：组织名称
3 LDAP 配置：DC 地址
4 共享配置：LDAP 服务器地址
5 共享配置：带URL 的 LDAP 服务器地址
6 共享配置：BaseDN 地址
7 共享配置：User DN 地址
8 共享配置：Group DN 地址

metadata:
  name: ldap-secret-admin
  namespace: core-system
  labels:
    type: ldap

stringData:
  password: admin (1)

apiVersion: v1
kind: Secret
metadata:
  name: ldap-secret-service
  namespace: core-system
  labels:
    type: ldap
  annotations: (2)
    reflector.v1.k8s.emberstack.com/reflection-allowed-namespaces: "kube-system,default,core-system,core-middleware,core-app,share-app,monitor-app"
    reflector.v1.k8s.emberstack.com/reflection-allowed: "true"
    reflector.v1.k8s.emberstack.com/reflection-auto-enabled: "true"
    reflector.v1.k8s.emberstack.com/reflection-auto-namespaces: "kube-system,default,core-system,core-middleware,core-app,share-app,monitor-app"
type: Opaque
stringData:
  user: 'service-account' (3)
  password: 'service-account' (4)
其中：
1 设置管理员密码
2 配置共享用户秘密共享
3 共享用户用户名
4 共享用户密码
生产环境安装一定要更改为复杂的密码！

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: ldap-system
  namespace: core-system
  labels:
    app: ldap-system
    type: ldap
spec:
  serviceName: ldap-system
  selector:
    matchLabels:
      app: ldap-system
  replicas: 1
  template:
    metadata:
      labels:
        app: ldap-system
        type: ldap
    spec:
      containers:
        - name: ldap-system
          image: osixia/openldap:1.5.0
          volumeMounts:
            - name: ldap-data
              mountPath: /var/lib/ldap
            - name: ldap-config
              mountPath: /etc/ldap/slapd.d
          ports:
            - containerPort: 389
              name: openldap
          env:
            - name: LDAP_LOG_LEVEL
              value: "256"
            - name: LDAP_ORGANISATION
              valueFrom:
                configMapKeyRef:
                  name: ldap-global-conf
                  key: ldap_org
            - name: LDAP_DOMAIN
              valueFrom:
                configMapKeyRef:
                  name: ldap-global-conf
                  key: ldap_domain
            - name: LDAP_ADMIN_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: ldap-secret-admin
                  key: password
            - name: LDAP_CONFIG_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: ldap-secret-admin
                  key: password
            - name: LDAP_READONLY_USER
              value: "true"
            - name: LDAP_READONLY_USER_USERNAME
              valueFrom:
                secretKeyRef:
                  name: ldap-secret-service
                  key: user
            - name: LDAP_READONLY_USER_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: ldap-secret-service
                  key: password
            - name: LDAP_RFC2307BIS_SCHEMA
              value: "false"
            - name: LDAP_BACKEND
              value: "mdb"
            - name: LDAP_TLS
              value: "false"
            - name: LDAP_REPLICATION
              value: "false"
            - name: KEEP_EXISTING_CONFIG
              value: "false"
            - name: LDAP_REMOVE_CONFIG_AFTER_SETUP
              value: "true"
      volumes:
        - name: ldap-data
          persistentVolumeClaim:
            claimName: pvc-ldap-data
        - name: ldap-config
          persistentVolumeClaim:
            claimName: pvc-ldap-config
---
apiVersion: v1
kind: Service
metadata:
  labels:
    app: service-ldap
    type: ldap
  name: service-ldap
  namespace: core-system
spec:
  ports:
    - port: 389
  selector:
    app: ldap-system

导入 MemberOf 配置

MemberOf 能非常方便的绑定用户和组的关系。进入 LDAP 控制台，执行以下命令：

cat << "EOF" > mem.ldif
dn: cn=module{0},cn=config
changetype: modify
add: olcModuleLoad
olcModuleLoad: memberof.la

dn: olcOverlay=memberof,olcDatabase={1}mdb,cn=config
objectClass: olcMemberOf
objectClass: olcOverlayConfig
objectClass: olcConfig
objectClass: top
olcOverlay: memberof
olcMemberOfRefInt: TRUE
olcMemberOfDangling: ignore
olcMemberOfGroupOC: groupOfNames
olcMemberOfMemberAD: member
olcMemberOfMemberOfAD: memberOf

dn: cn=module{0},cn=config
changetype: modify
add: olcModuleLoad
olcModuleLoad: refint.la
EOF
ldapadd -Y EXTERNAL -H ldapi:/// -f ./mem.ldif

验证

部署完成后，执行以下命令验证部署的完整性。

1
2
3

# 检查配置是否同步到 default 命名空间
kubectl get secrets,configmaps  -n core-system
kubectl get pods,service -n core-system

确认 Pod 状态均为 Running 且未多次重启即表示安装无误。
